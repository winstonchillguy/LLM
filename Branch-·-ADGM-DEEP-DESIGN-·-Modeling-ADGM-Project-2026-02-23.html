✅ SAFE-TO-ARM + FIRE LOGIC (FULL STATE MACHINE)

🎯 Design Objective

The fuse system must ensure that:

  1. The munition cannot arm in storage/handling.

  2. The munition cannot arm in accidental drop/impact.

  3. The munition cannot arm if launched incorrectly.

  4. The munition cannot fire unless:

    • it is armed,

    • target is confirmed,

    • geometry is correct,

    • spin phase alignment is achieved,

    • capacitor is charged,

    • all safety windows are satisfied.

This is implemented as a multi-condition arming gate + fire authorization gate.

1. SYSTEM SIGNAL INPUTS (WHAT THE LOGIC USES)

A. Sensors / Inputs

  • Setback Acceleration Sensor (high-G accelerometer or IMU accel)

  • Spin Rate Measurement (gyro or accelerometer phase signature)

  • Time Since Launch Clock (internal timer)

  • MMIC Radar Return Stream

  • ML Inference Output

  • Temperature Sensor (optional but recommended)

  • Battery Voltage / Power Health

  • Capacitor Charge Voltage

  • MCU Watchdog / Self-check flag

B. Required Computed Variables

  • t_launch = time since detected launch event

  • ω_spin = spin rate in rad/s or RPM

  • θ_phase = angular phase of the projectile (0–360°)

  • TargetConfidence = ML probability score

  • TargetClass = {Class0, Class1, Class2, Class3}

  • TargetVector = predicted 3D direction of target relative to projectile

  • DistanceEstimate = inferred range to target (radar based)

  • ApproachMode = {HeadOn, Crossing, Receding, Unknown}

2. SAFE-TO-ARM PRINCIPLE (MULTI-LAYER GATING)

Your fuze arms ONLY if all of these are true:

Layer 1: Launch Event Validation

  • A setback impulse is detected exceeding threshold:

    • Example: Accel > 8,000g for > 0.5 ms

Layer 2: Spin Validation

  • Confirm rotation exceeds minimum spin:

    • Example: SpinRate > 5,000 RPM sustained for > 2 ms

Layer 3: Flight Continuity Check

  • Confirm projectile is in continuous free-flight:

    • no “impact signature” after setback

    • acceleration stabilizes into ballistic pattern

Layer 4: Time Window Gate

  • Arming cannot happen immediately.

  • Typical requirement:

    • minimum safe separation from shooter.

Example:

  • ARM_ENABLE_TIME = 20 ms after launch

  • ARM_ABORT_TIME = 5 seconds (if no target, self-disarm)

Layer 5: Power Integrity Gate

  • MCU must verify:

    • battery voltage OK

    • regulator stable

    • capacitor charging circuit functioning

    • no memory corruption

If any layer fails → system stays inert.

3. FULL STATE MACHINE (IMPLEMENTATION READY)

STATE 0 — STORAGE / INERT (Default)

System Condition:

  • No launch detected

  • Capacitor discharge path physically blocked

  • EFI trigger line disabled

  • Radar may be off or low-power mode

Allowed transitions:

STATE 1 — SETBACK CONFIRMATION (Launch Validation)

Entry condition:

  • High-G acceleration spike detected

Actions:

  • Begin high-frequency IMU sampling

  • Power rails fully enabled

  • Radar warms up / boot begins

Validation conditions:

  • Must confirm it is a true gun-launch signature:

Transition:

  • If invalid → return to STATE 0

Purpose: Ensure projectile is actually rifled and stable.

Actions:

  • Estimate spin phase tracking stability

  • Confirm no abnormal tumbling

Conditions required:

  • SpinRate > ω_min continuously for X milliseconds

  • SpinStability == TRUE (gyro waveform consistent)

Transition:

  • If valid → STATE 3

Purpose: Prevent detonation close to the barrel/platform.

Actions:

  • Continue capacitor charging

Condition:
